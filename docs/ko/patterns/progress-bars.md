# 프로그레스바

JSON UI 프로그레스바는 보통 테두리, 잘라내기 패널, 채움 이미지로 만듭니다.

<img class="jsonui-diagram" src="/assets/diagrams/progress-clip.svg" alt="프로그레스바 잘라내기 패턴">

## 잘라내기 기반 바

가장 안정적인 구조:

```text
bar_frame
  clip_panel     <- 이 너비가 진행도를 의미
    fill_image   <- 더 크거나 전체 너비의 텍스처
```

테두리는 고정하고, 잘라내기 패널의 보이는 너비만 바꿉니다.

바로 복사해서 볼 수 있는 예제는 [예제 코드](../examples/index.md#프로그레스바)에 넣어 두었습니다.

## 데이터 소스

프로그레스바는 아래 값으로 구동할 수 있습니다.

- HUD에 노출된 스코어보드 값
- title/actionbar로 보낸 데이터
- 서버폼의 `#form_text` 페이로드
- 커스텀 폼 텍스트
- 실제 값 없이 움직이기만 하는 로딩 상태

## 서버폼 진행도

서버폼에서는 서버가 본문 텍스트에 진행도 값을 인코딩할 수 있습니다.

```text
progress:75|Mining Level
```

JSON UI는 이 값을 잘라내기 폭으로 바꾸거나, 미리 준비한 단계 중 하나를 선택하게 만들 수 있습니다. 단순한 메뉴라면 임의의 픽셀 폭을 계산하려고 하기보다 0%, 25%, 50%, 75%, 100%처럼 단계를 나누는 편이 훨씬 안정적입니다.

## 움직이는 로딩 바

실제 값이 없으면 indeterminate 애니메이션을 사용하세요.

```json
{
  "anim_type": "offset",
  "duration": 1.2,
  "from": [-80, 0],
  "to": [220, 0],
  "play_event": "always",
  "loop": true
}
```

이 방식은 실제 진행도가 아니라 대기 상태 표시에 적합합니다.

## 아이콘 진행도

하트나 배고픔 UI 같은 진행도는 보통 반복 아이콘과 단계별 표시 조건으로 만듭니다.

```text
1번 조각: 값이 1 이상이면 표시
2번 조각: 값이 2 이상이면 표시
3번 조각: 값이 3 이상이면 표시
```

아이콘 텍스처를 억지로 늘리는 것보다 예측하기 쉽습니다.

## 디버깅

바가 보이지 않으면:

1. frame과 fill 텍스처를 따로 확인합니다.
2. 잘라내기를 잠시 끕니다.
3. 원본 값을 디버그용 라벨에 출력합니다.
4. 부모 패널 크기가 0이 아닌지 확인합니다.
