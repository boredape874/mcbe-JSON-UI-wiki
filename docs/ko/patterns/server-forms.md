# 서버폼 패턴

이 문서는 서버폼 기본 흐름을 이해한 뒤 바로 재사용할 수 있는 실전 패턴을 정리합니다.

## 얇은 라우터 패턴

`server_form.json`은 화면 진입과 라우팅만 담당하게 하세요. 기능별 실제 UI는 별도 폴더로 나눕니다.

```text
ui/server_form.json
ui/forms/shop_menu/index.json
ui/forms/shop_menu/templates.json
ui/forms/admin_console/index.json
ui/forms/traits/index.json
```

이 구조가 큰 서버 UI 팩을 가장 오래 버티게 해줍니다.

## 제목 토큰 패턴

서버가 제목에 안정적인 마커를 붙입니다.

```text
[shop] Shop Menu
[admin] Admin Console
```

리소스팩은 이 마커를 보고 어떤 커스텀 레이아웃을 보여줄지 결정합니다. 플레이어에게 보이는 제목은 별도 라벨로 다시 그리면 되므로, 마커를 그대로 노출할 필요는 없습니다.

## 컬렉션 버튼 패턴

서버가 보낸 액션 버튼은 `form_buttons` 컬렉션으로 렌더링합니다. 버튼 템플릿은 재사용 가능하게 만들고, 템플릿 내부에서 `collection_details` 바인딩을 연결합니다.

적합한 용도:

- 상점 카테고리 카드
- 워프 메뉴
- 관리자 액션 패널
- 특성 또는 스킬 노드
- 이미지 기반 메뉴 타일

## 검색 바 패턴

서버폼 검색 UI는 보통 아래 요소를 조합합니다.

- 입력용 `common.text_edit_box`
- 입력값을 검색 프로퍼티로 복사하는 view binding
- 각 버튼의 `#form_button_text`와 검색 문자열을 비교하는 visible 조건

기본 비교는 대소문자를 구분합니다. 실제 메뉴에서는 서버가 소문자 검색 키를 따로 보내는 방식이 UI 쪽 문자열 처리를 줄일 수 있어 안정적입니다.

## 잠긴 노드 패턴

특성 트리, 퀘스트, 성장 메뉴에서는 아래 방식이 좋습니다.

1. 모든 노드는 서버 버튼 인덱스를 유지합니다.
2. JSON UI는 아이콘과 자물쇠 오버레이를 그립니다.
3. 잠긴 노드를 클릭하면 서버가 최종적으로 거부합니다.
4. 잠금 해제 후에는 서버가 폼을 다시 열어 자물쇠를 제거합니다.

UI는 상태를 보여주는 역할이고, 서버는 상태를 강제하는 역할입니다.

## 텍스처 기반 패턴

고퀄리티 폼은 복잡한 장식을 JSON 패널 여러 개로 쌓기보다 텍스처로 처리하는 편이 안정적입니다.

주로 텍스처로 빼기 좋은 요소:

- 패널 배경
- 카드 배경
- hover 프레임
- pressed 상태
- 자물쇠 오버레이
- 아이콘 슬롯

JSON 컨트롤은 배치와 상태 처리에 쓰고, 그라데이션, 픽셀 테두리, 그림자 같은 시각 요소는 텍스처에 맡기는 것이 유지보수에 유리합니다.
