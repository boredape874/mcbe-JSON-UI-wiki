# 서버폼 패턴

이 문서는 서버폼의 기본 흐름을 이해한 뒤 재사용할 수 있는 실전 패턴을 정리합니다.

## 얇은 라우터 패턴

`server_form.json`은 화면 진입과 라우팅에만 사용하고, 기능별 UI는 각자 폴더에 둡니다.

```text
ui/server_form.json
ui/forms/shop_menu/index.json
ui/forms/shop_menu/templates.json
ui/forms/admin_console/index.json
ui/forms/traits/index.json
```

이 구조를 쓰면 서버 UI 팩이 커져도 읽기 쉽습니다.

## 제목 토큰 패턴

서버가 폼 제목에 안정적인 마커를 붙입니다.

```text
[shop] Shop Menu
[admin] Admin Console
```

리소스팩은 이 마커를 보고 어떤 커스텀 레이아웃을 보여줄지 결정합니다. 플레이어에게 보이는 제목은 별도 라벨로 깔끔하게 다시 그릴 수 있으므로, 마커를 그대로 노출할 필요는 없습니다.

## 컬렉션 버튼 패턴

서버가 제공한 액션 버튼은 `form_buttons` 컬렉션으로 렌더링합니다. 버튼 템플릿은 재사용 가능하게 만들고, 템플릿 안에서 컬렉션 상세 바인딩을 연결합니다.

잘 맞는 용도:

- 상점 카테고리 카드
- 워프 메뉴
- 관리자 액션 패널
- 특성 또는 스킬 노드
- 이미지 기반 메뉴 타일

## 검색 바 패턴

서버폼 검색 UI는 보통 아래 요소를 조합합니다.

- 입력용 `common.text_edit_box`
- 입력 값을 검색용 속성으로 복사하는 view 바인딩
- 각 버튼의 `#form_button_text`와 검색 문자열을 비교하는 visible 조건

검색은 별도 정규화 레이어가 없으면 대소문자를 구분합니다. 실제 서비스 메뉴에서는 서버가 소문자 검색 키를 함께 보내는 편이 UI 쪽 문자열 처리를 줄일 수 있습니다.

## 잠긴 노드 패턴

스킬 트리, 퀘스트, 성장 메뉴에서는 아래 구조가 안전합니다.

1. 모든 노드를 일반 서버 버튼 인덱스로 유지합니다.
2. JSON UI에서 아이콘과 자물쇠 오버레이를 그립니다.
3. 잠긴 노드 클릭은 서버가 거부합니다.
4. 잠금 해제 후 다음 폼 새로고침에서 자물쇠를 제거합니다.

UI는 상태를 보여주고, 서버는 상태를 강제해야 합니다.

## 텍스처 기반 패턴

완성도 높은 폼은 아래 요소를 커스텀 텍스처로 처리하는 경우가 많습니다.

- 패널 배경
- 카드 배경
- hover 프레임
- pressed 상태
- 자물쇠 오버레이
- 아이콘 슬롯

JSON 컨트롤은 배치와 상태 처리에 쓰고, 복잡한 그라데이션, 픽셀 아트 테두리, 장식 그림자는 텍스처로 빼는 편이 안정적입니다.
