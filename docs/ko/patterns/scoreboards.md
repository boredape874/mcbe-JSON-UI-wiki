# 스코어보드

스코어보드 UI는 서버가 플레이어별 값을 이미 scoreboard로 노출하고 있을 때 쓰기 좋습니다.

## 잘 맞는 용도

- 작은 스탯 패널
- 랭크나 재화 표시
- 퀘스트 카운터
- 서버 상태 오버레이
- 사이드바 대체 UI

복잡한 상호작용 메뉴에는 맞지 않습니다. 스코어보드는 입력보다 표시용에 가깝습니다.

## 구조

```text
scoreboard data
        -> scoreboard screen 또는 HUD insert
        -> label binding
        -> row, icon, separator 스타일링
```

## 디자인 조언

스코어보드 패널은 작고 읽기 쉬워야 합니다. 작은 아이콘, 정렬된 숫자, 고정된 row 높이를 사용하세요. 게임 화면을 가리는 큰 장식 배경은 피하는 편이 좋습니다.

## 디버깅

값이 갱신되지 않으면:

1. 게임 안에서 scoreboard 값이 실제로 바뀌는지 확인합니다.
2. 올바른 scoreboard UI 파일을 수정하고 있는지 확인합니다.
3. 스타일링 전에 원본 텍스트를 임시로 출력합니다.
4. 다른 리소스팩이 같은 화면을 덮어쓰고 있지 않은지 확인합니다.
