# JSON UI란?

Minecraft Bedrock JSON UI는 리소스팩에서 클라이언트 UI를 정의하는 시스템입니다. 화면, 패널, 라벨, 이미지, 버튼, 서버폼, HUD 위젯 같은 요소를 JSON 파일로 설명합니다.

대부분의 UI 구조가 JSON 객체로 작성되기 때문에 JSON UI라고 부릅니다.

```json
{
  "type": "label",
  "text": "Hello JSON UI",
  "size": [120, 20],
  "anchor_from": "center",
  "anchor_to": "center"
}
```

이 작은 객체는 "라벨을 만들고, 텍스트를 넣고, 크기를 지정한 뒤 중앙에 배치하라"는 뜻입니다.

## 핵심 개념

JSON UI는 컨트롤 트리입니다.

<img class="jsonui-diagram" src="/assets/diagrams/control-tree.svg" alt="JSON UI 컨트롤 트리">

화면은 여러 컨트롤을 포함하고, 컨트롤은 다시 다른 컨트롤을 포함할 수 있습니다.

- 화면 패널 안에 배경 이미지가 들어갑니다.
- 같은 패널 안에 버튼이 들어갑니다.
- 버튼 안에는 텍스트 라벨이 들어갈 수 있습니다.
- 버튼은 hover나 pressed 상태에 따라 다른 자식 컨트롤을 보여줄 수 있습니다.

## JSON UI로 바꿀 수 있는 것

JSON UI로 여러 베드락 클라이언트 화면을 커스터마이징할 수 있습니다.

- HUD 오버레이
- 채팅 패널
- 타이틀과 액션바 표시
- 서버폼
- 인벤토리와 비슷한 화면
- 시작 화면과 일시정지 화면 일부
- 스코어보드, 타이틀, 폼 데이터로 움직이는 커스텀 시각 레이어

## JSON UI만으로 할 수 없는 것

JSON UI는 스크립트 언어가 아닙니다.

직접 할 수 없는 일:

- 서버 명령 실행
- 플레이어 데이터 저장
- 블록 이벤트 감지
- 게임플레이 로직 생성

게임 로직은 서버, BP Script, PMMP 플러그인에서 처리하고 JSON UI는 클라이언트 화면 표시를 맡습니다.

## 입문자 규칙

JSON UI는 이렇게 이해하면 됩니다.

> 베드락 클라이언트에게 "이 화면을 어떻게 보여주고, 이미 있는 UI 데이터에 어떻게 반응할지" 알려주는 리소스팩 파일.

이 관점으로 보면 초반 혼란이 많이 줄어듭니다.
