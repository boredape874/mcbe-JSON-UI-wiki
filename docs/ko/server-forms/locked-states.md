# 잠금 상태

잠금 상태는 특성 트리, 상점, 퀘스트, 보상, 키트, 관리자 패널에서 자주 쓰입니다.

중요한 규칙:

> JSON UI는 잠금을 보여줄 수 있지만, 잠금을 강제하는 것은 서버여야 합니다.

## 추천 흐름

```text
서버가 모든 노드를 포함한 폼을 보냄
        -> JSON UI가 노드 아이콘을 그림
        -> 잠긴 노드에는 자물쇠 오버레이를 표시
        -> 플레이어가 노드를 클릭
        -> 서버가 잠금 해제 상태를 검사
        -> 서버가 수락, 거부, 또는 폼 새로고침
```

## 버튼 인덱스 안정성

모든 인덱스를 다시 매핑할 준비가 되어 있지 않다면 잠긴 버튼을 컬렉션에서 제거하지 마세요.

더 좋은 방식:

- 버튼 인덱스를 유지
- 자물쇠 오버레이 표시
- 필요하면 아이콘을 어둡게 표시
- 서버가 클릭을 거부

## 페이로드 패턴

UI가 상태를 보여줘야 한다면 서버가 제목, 본문, 버튼 텍스트에 상태를 인코딩할 수 있습니다.

```text
button 7 text: trait:combat_crit|locked|Critical Strike
button 8 text: trait:combat_speed|unlocked|Rush
```

UI는 마커를 파싱해 표시를 바꾸고, 실제 잠금 상태는 계속 서버가 소유합니다.

## 시각 패턴

잠긴 노드는 보통 아래 요소를 가집니다.

- 기본 슬롯 텍스처
- 아이콘 텍스처
- 어두운 오버레이
- 자물쇠 아이콘
- 선택적 요구 조건 라벨

자물쇠 아이콘과 복잡한 그림자는 텍스처로 처리하고, JSON 컨트롤은 상태, 위치, visible 제어에 사용하세요.

## 흔한 실수

버튼을 숨기는 것을 보안으로 생각하지 마세요. 숨겨져 있거나 비활성화처럼 보이는 UI 요소는 권한 시스템이 아닙니다. 서버 응답 핸들러에서 반드시 상태를 다시 확인해야 합니다.
